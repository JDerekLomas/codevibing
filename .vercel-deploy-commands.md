# Quick Vercel Deploy Commands

## First Time Setup

```bash
# Install Vercel CLI globally
npm install -g vercel

# Login to your Vercel account
vercel login

# Navigate to project directory
cd /Users/dereklomas/codevibing/codevibing
```

## Deploy

```bash
# Deploy to preview (development)
vercel

# Deploy to production
vercel --prod

# Deploy with specific environment
vercel --env production
```

## Environment Variables

```bash
# Add a new environment variable
vercel env add ANTHROPIC_API_KEY

# List all environment variables
vercel env ls

# Pull environment variables to local
vercel env pull

# Remove an environment variable
vercel env rm ANTHROPIC_API_KEY
```

## Project Management

```bash
# Link to existing Vercel project
vercel link

# Show project info
vercel inspect

# List all deployments
vercel ls

# View deployment logs
vercel logs [deployment-url]

# Remove a deployment
vercel rm [deployment-url]
```

## Domain Management

```bash
# Add a domain
vercel domains add example.com

# List all domains
vercel domains ls

# Remove a domain
vercel domains rm example.com
```

## Local Development

```bash
# Run development server (Next.js)
npm run dev

# Build for production locally
npm run build

# Start production server locally
npm run start

# Verify environment variables
npm run verify-env

# Pre-deployment check (lint + build + env check)
npm run predeploy
```

## Troubleshooting

```bash
# View build logs
vercel logs --follow

# Inspect specific deployment
vercel inspect [url]

# Rollback to previous deployment
# (Deploy the old deployment URL again)
vercel --prod [previous-deployment-url]

# Clear local credentials
vercel logout
```

## Useful Flags

```bash
--prod                 # Deploy to production
--force                # Force a new deployment
--public               # Make deployment public
--env KEY=value        # Set environment variable inline
--regions iad1         # Specify deployment region
--yes                  # Skip confirmation prompts
```

## Quick Deploy Workflow

```bash
# 1. Make changes to code
git add .
git commit -m "Your changes"

# 2. Test locally
npm run predeploy

# 3. Push to Git (triggers automatic deploy if configured)
git push origin main

# OR deploy manually
vercel --prod
```

## Emergency Rollback

```bash
# 1. List recent deployments
vercel ls

# 2. Find working deployment URL
# 3. Promote it to production
vercel --prod https://codevibing-abc123.vercel.app

# OR use alias
vercel alias https://codevibing-abc123.vercel.app codevibing.vercel.app
```

## Monitoring

```bash
# Real-time function logs
vercel logs --follow

# Specific deployment logs
vercel logs [deployment-url]

# Check deployment status
vercel inspect [deployment-url]
```

## Pro Tips

1. Use `vercel env pull` to sync environment variables locally
2. Preview deployments get unique URLs automatically for each branch/PR
3. Use `vercel --prod` only for production deployments
4. Set up Git integration for automatic deployments
5. Use `vercel alias` to manage custom domains
