# Flourish Agent Explorer

Interactive demo that helps product teams brainstorm agents aimed at human flourishing, informed by Pieter Desmet's 13 fundamental needs. The frontend lets users pick a need, describe their design scenario, and receive tailored agent concepts generated by OpenAI's Responses API.

## Getting started

1. Install dependencies:
   ```bash
   npm install
   ```
2. Provide your OpenAI API key:
   ```bash
   cp .env.example .env
   # edit .env and add OPENAI_API_KEY
   ```
3. Run the server:
   ```bash
   npm run dev
   ```
4. Open [http://localhost:3000](http://localhost:3000) to explore. If you run on a restricted environment, set `HOST=127.0.0.1` (or your desired bind address) in `.env`.

From the UI, select a Desmet need, describe the **application domain**, and add your design brief—the ChatGPT backend now returns a creativity briefing (theme + imagination prompts), a design-space analysis (agent capability clusters + key AgentKit affordances), and compact Markdown cards wrapped in `<!-- card:start -->` / `<!-- card:end -->` markers. Each card surfaces motivation, a short description, workflow bullets, and a quick next experiment, and you can leave thumbs-up or thumbs-down reactions for triage. Toggle evaluator agents (safety, empathy, evidence, bias, delight) to receive tailored evaluation plans you can activate in your workflow.

When the OpenAI API is unreachable, the server returns sample concepts sourced from a built-in library so the interface remains usable. Check your server logs for the underlying error (for example, missing API key or blocked network) and retry once resolved.

## Environment variables

Create `.env` with:

```
OPENAI_API_KEY=sk-...
PORT=3000
```

## Project structure

- `server.js` – Express server exposing REST endpoints and proxying brainstorm requests to OpenAI.
- `public/` – Static frontend assets (HTML, CSS, JavaScript) served by the Express app.

## Extending the demo

- Swap `gpt-4.1-mini` for a larger model (for example `o4-mini`) or upgrade to Agents SDK for multi-tool workflows.
- Introduce persistent conversation state (sessions) or user auth.
- Layer in evaluation workflows with OpenAI Evals, or add Guardrails policies for safe deployment.
- Connect additional data sources using AgentKit's Connector Registry once available.
- Optional: configure Supabase (set `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY`) and create an `idea_votes` table with columns like `client_id text`, `idea_id text`, `idea_title text`, `idea_data jsonb`, `needs text`, `domain text`, `design_context text`, `evaluators jsonb`, plus timestamps. This stores thumbs-up ideas while thumbs-down removes them.
